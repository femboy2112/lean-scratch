# Feature Completeness Matrix - v0.4

This matrix answers the release-audit question: have all TLICA features been
mathematically explained?

Answer: no. The project has a verified foundation spine and first structural
kernels for agency, free will, deterministic temporal trajectory, and
differentiated affect. Several features remain partial, deferred, or prose-only.

## Status Categories

| Category | Meaning |
|---|---|
| `fully_machine_formalized` | The feature has imported Lean definitions and the expected internal theorem coverage for the current mathematical claim. |
| `partially_machine_formalized` | Core definitions and some consequences are imported, but intended theorems or application semantics remain open. |
| `structure_only` | Lean structure/signature exists, with assumptions carried as fields and little or no derived theorem coverage. |
| `signature_only` | Lean declaration fixes a name or predicate shape but does not yet carry substantial content. |
| `deferred_marker_only` | Lean has only an explicit `..._deferred : Prop := True` marker. This is not theorem evidence. |
| `prose_only_unformalized` | No Lean declaration exists. |
| `compatibility_only` | Name exists only for API/backward compatibility over a primary object. |
| `deprecated_or_superseded` | Historical name or broad target superseded by normalized rows. |
| `needs_source_refinement_before_lean` | The prose/math object is not precise enough for Lean. |
| `lean_ready_next_target` | The next theorem or structure target appears low enough risk for a focused Lean branch. |
